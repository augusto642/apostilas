Módulo 1

Panorama da web e arquitetura básica

Antes de escrever código, é importante entender o ambiente em que um sistema web funciona.

🧭
Ideia central Internet é a infraestrutura de redes. Web é um dos serviços que usam essa infraestrutura, baseada em páginas, links, protocolos e navegadores.

1.1 Como a web funciona

Quando uma pessoa acessa um site, o navegador faz uma requisição para um servidor usando protocolos como HTTP ou HTTPS. O servidor devolve arquivos como HTML, CSS, JavaScript, imagens e fontes. O navegador interpreta tudo isso, monta a árvore DOM, aplica estilos, executa scripts e apresenta a interface.

Front-end

Camada visível ao usuário. Trabalha com interface, usabilidade, responsividade e interação.

HTML CSS JavaScript

Back-end

Camada de regras de negócio, autenticação, banco de dados e serviços de API.

Node.js Python SQL

Full stack

Profissional que entende a comunicação entre interface, servidor, versionamento e deploy.

1.3 Competências profissionais da web moderna

Qualidade visual e semântica

O mercado espera páginas com hierarquia clara, conteúdo bem escrito e HTML capaz de ser entendido por pessoas, buscadores e tecnologias assistivas.

Acessibilidade e inclusão

Contraste adequado, navegação por teclado, textos alternativos e formulários bem rotulados deixaram de ser diferencial e passaram a ser requisito profissional.

Desempenho e experiência

Projetos web modernos precisam carregar rápido, funcionar no celular e evitar excesso de imagens pesadas, scripts desnecessários e navegação confusa.

Organização de projeto

Mesmo em projetos simples, saber nomear arquivos, separar responsabilidades e publicar com versionamento mostra maturidade técnica.

Atividade prática: mapear a jornada de uma página

  1. Escolha um site conhecido da escola, da prefeitura ou de uma empresa local.
  2. Identifique onde aparecem HTML, CSS e JavaScript na experiência do usuário.
  3. Descreva uma requisição simples: URL, navegador, resposta do servidor e interface exibida.
  4. Registre dois pontos fortes e duas oportunidades de melhoria no site analisado.

1.4 Rota de aprendizagem do zero ao primeiro portfólio

Uma progressão eficiente para turmas técnicas começa em visão de internet, passa por HTML e CSS com exemplos pequenos e só depois avança para publicação e projeto autoral. Nesta revisão, a trilha foi reforçada com temas recorrentes do repositório aberto gustavoguanabara/html-css, como história da internet, primeiros passos em HTML, listas, links, imagens, estilos, cores, fontes, box model, mini projetos e GitHub.

Fase 1

Entender a web antes do código

O estudante precisa dominar o cenário em que a página existe.

  • história e funcionamento da internet;

Antes de pensar em efeitos, a página precisa fazer sentido sem CSS.

  • títulos, parágrafos, listas, links e imagens;

  • organização de arquivos, nomes e navegação interna.
Fase 3

Dar identidade visual ao projeto

Com a base pronta, entra o trabalho de layout e leitura visual.

  • cores, fontes, seletores e box model;
  • cards, menus, botões, responsividade e hierarquia;
  • uso criterioso de imagens de fundo e efeitos.
Fase 4

Transformar estudo em entrega

O conteúdo só se consolida quando vira portfólio e publicação.

  • mini projetos com contexto real;
  • versionamento com Git e GitHub;
  • publicação online e revisão final de qualidade.

Exercícios do módulo

1. Diferencie internet, web e site.

Internet é a infraestrutura global de redes. Web é um serviço que usa essa infraestrutura para entregar páginas e recursos interligados. Site é um conjunto organizado de páginas publicadas na web.

2. Cite a função principal de HTML, CSS e JavaScript em uma página.

HTML organiza a estrutura, CSS define a aparência visual e JavaScript adiciona comportamento e lógica.

3. Em uma requisição web simples, quem envia a solicitação e quem devolve a resposta?

O navegador envia a solicitação ao servidor, e o servidor devolve a resposta com arquivos ou dados.
Módulo 2

HTML: estrutura, semântica e acessibilidade

HTML não é “apenas texto com tags”; ele descreve o significado do conteúdo e ajuda tecnologias assistivas a interpretar a página.

🏗️
Regra de ouro Use a tag que representa melhor o conteúdo. O navegador pode até “aceitar” qualquer marcação, mas semântica correta melhora manutenção, acessibilidade e busca.

2.1 Esqueleto de uma página HTML

<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Minha primeira página</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header>
    <h1>Loja Escola</h1>
  </header>

  <main>
    <section>
      <h2>Produtos em destaque</h2>
      <p>Conteúdo principal da página.</p>
    </section>
  </main>

  <script src="app.js"></script>
</body>
</html>

2.2 Tags semânticas mais importantes

Tag Uso recomendado Evite usar quando...
<header> Cabeçalho de página ou seção for apenas um contêiner visual sem valor semântico
<nav> Conjunto principal de links de navegação houver links dispersos sem função de menu
<main> Conteúdo principal único da página existirem várias áreas principais concorrentes
<section> Bloco temático com título for só um agrupamento genérico sem assunto
<article> Conteúdo independente, como notícia ou post o trecho só fizer sentido dentro do contexto maior
<aside> Conteúdo complementar, dicas, widgets o bloco for parte central do fluxo
<footer> Rodapé da página ou de uma seção for apenas um bloco visual aleatório

2.3 Acessibilidade desde o início

  • Defina o idioma da página com lang.
  • Use hierarquia coerente de títulos: h1, h2, h3...
  • Associe rótulos a campos de formulário com <label for="...">.
  • Forneça texto alternativo útil em imagens com alt.
  • Garanta contraste adequado entre texto e fundo.
  • Evite depender apenas de cor para indicar erro ou sucesso.

2.4 Imagens, mídias e conteúdo incorporado

Recurso Quando faz sentido Boa prática essencial

PNG Imagens com transparência e elementos gráficos simples Evitar arquivos maiores do que o necessário
WebP Imagens modernas com bom equilíbrio entre qualidade e tamanho Usar sempre que houver suporte no projeto
SVG Ícones, logotipos e ilustrações vetoriais Preferir para elementos que precisam escalar sem perder qualidade

🖼️
Ponto importante Além de saber inserir arquivos, o estudante precisa observar direitos de uso de imagens, nomear arquivos com clareza, criar favicons e preferir mídias leves para melhorar o carregamento.

2.5 Tabelas e organização de dados

Tabelas HTML servem para dados tabulares: horários, notas, estoque, cronogramas e comparações. Não devem ser usadas para montar layout da página.

<table>
  <caption>Horários do laboratório</caption>
  <thead>
    <tr>
      <th scope="col">Turma</th>
      <th scope="col">Dia</th>
      <th scope="col">Horário</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Informática A</td>
      <td>Segunda</td>
      <td>19h às 20h40</td>
    </tr>
  </tbody>
</table>
  • Use <caption> para dar contexto ao conteúdo.
  • Prefira <th scope="col"> e <th scope="row"> para facilitar leitura por tecnologias assistivas.
  • Em telas pequenas, pense em rolagem horizontal ou reorganização visual, em vez de reduzir demais o texto.

2.6 Fundamentos de HTML que separam página improvisada de material profissional

Textos, caracteres e ênfases

Um iniciante precisa saber quando usar parágrafo, quebra de linha, citação, ênfase e entidades HTML. Isso evita páginas visualmente confusas e melhora a leitura.

  • prefira <p> para blocos de texto em vez de empilhar <br>;
  • use &amp;, &lt; e outras entidades quando o conteúdo exigir;
  • reserve <strong> e <em> para dar significado, não apenas aparência.

Listas que ajudam a navegar

Listas ordenadas e não ordenadas aparecem em menus, passos, glossários, checklists e seções de conteúdo. Elas organizam informação melhor do que blocos soltos.

  • use <ol> quando a sequência importa;
  • use <ul> para agrupamentos sem ordem rígida;
  • transforme listas em menus com CSS nas próximas etapas.

Links que orientam a experiência

Uma página madura informa para onde leva, o que abre e em que contexto o usuário está navegando. Bons links economizam tempo e reduzem erros.

  • escreva textos de link com significado real;
  • use âncoras internas para páginas extensas;
  • indique claramente downloads, sites externos e contatos.

Favicons, pastas e organização

Detalhes simples passam sensação de projeto completo: favicon, nomes coerentes, pastas previsíveis e mídia otimizada.

  • separe imagens, estilos e scripts quando o projeto crescer;
  • padronize nomes de arquivos com clareza;
  • inclua favicon e títulos de página pensados para navegação real.
<nav aria-label="Navegação principal">
  <ul>
    <li><a href="#sobre">Sobre o projeto</a></li>
    <li><a href="#oficinas">Oficinas</a></li>
    <li><a href="#contato">Contato</a></li>
  </ul>
</nav>

<link rel="shortcut icon" href="favicon.ico" type="image/x-icon">

1. Por que usar <main> é melhor do que usar apenas <div> para o conteúdo central?

Melhor acessibilidade, melhor SEO, organização mais clara do código e manutenção mais simples em equipe.
Módulo 3

CSS: estilo, layout e design responsivo

CSS transforma uma estrutura simples em uma interface legível, agradável e adaptável a diferentes telas.

🎨
CSS não é “decoração” Em projetos profissionais, CSS também organiza informação, reforça hierarquia visual, melhora legibilidade e ajuda a construir experiência de uso.

3.1 Conceitos essenciais

Cascata

Quando várias regras atingem o mesmo elemento, o navegador decide qual aplicar considerando origem, especificidade e ordem.

Especificidade

Seletores mais específicos tendem a vencer seletores mais genéricos. IDs têm peso maior que classes, que têm peso maior que tags.

Box model

Todo elemento é uma caixa com conteúdo, preenchimento, borda e margem.

Responsividade

A interface deve funcionar em diferentes tamanhos de tela usando media queries, unidades relativas e layouts fluidos.

3.2 Seletores, unidades e layout

Recurso Exemplo Quando usar
Classe .card Estilos reutilizáveis em vários elementos
ID #menu Elemento único ou alvo de navegação
Unidade relativa rem, %, vw Interfaces adaptáveis e acessíveis
Flexbox display:flex Alinhar itens em linha ou coluna
Grid display:grid Layouts bidimensionais mais controlados
Media query @media (max-width: 768px) Ajustes por faixa de tela 
:root {
  --cor-primaria: #2563eb;
  --espaco: 1rem;
}

.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: var(--espaco);
}

.card {
  padding: 1rem;
  border-radius: 12px;
  background: white;
  box-shadow: 0 10px 24px rgba(0, 0, 0, 0.08);
}

@media (max-width: 768px) {
  body {
    font-size: 15px;
  }
}

3.3 Cores, tipografia e efeitos que elevam a interface

Cores e contraste

Além de hexadecimal, CSS aceita rgb(), rgba(), hsl() e gradientes. Escolher paletas coerentes melhora identidade visual e leitura.

Tipografia

Fontes definem tom e legibilidade. É comum combinar uma fonte para títulos e outra para texto corrido, usando serviços como Google Fonts ou arquivos locais com @font-face.

Pseudo-classes e pseudo-elementos

:hover, :focus e :nth-child() ajudam na interação e organização visual. Já ::before e ::after servem para detalhes decorativos sem poluir o HTML.

Fundos, transições e microefeitos

Planos de fundo, sombras, transições suaves e pequenas animações podem destacar elementos importantes sem exagero. O foco deve continuar sendo clareza e usabilidade.

@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap');

body {
  font-family: 'Inter', sans-serif;
  background-image: linear-gradient(180deg, #eff6ff, #ffffff);
}

.btn {
  transition: transform .2s ease, background-color .2s ease;
}

.btn:hover {
  transform: translateY(-2px);
}

.lista li:nth-child(odd) {
  background: rgba(37, 99, 235, 0.06);
}

3.4 Flexbox, Grid e animações modernas

Técnica Propriedades centrais Cenário comum
Flexbox justify-content, align-items, gap, flex-wrap Menus, grupos de botões, cartões em linha e alinhamento vertical
Grid grid-template-columns, grid-template-areas, fr, place-items Dashboards, galerias, páginas com várias regiões e layouts mais rígidos
Transições transition, duration, timing-function Hover em botões, foco de campos, menus e cartões
Animações @keyframes, animation-name, animation-duration Entradas suaves, indicadores de carregamento e destaques visuais 
.galeria {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.dashboard {
  display: grid;
  grid-template-columns: 2fr 1fr;
  gap: 1rem;
}

@keyframes surgir {
  from { opacity: 0; transform: translateY(12px); }
  to { opacity: 1; transform: translateY(0); }
}

.painel {
  animation: surgir .4s ease-out;
}

Critério de qualidade: efeitos devem comunicar estado ou reforçar hierarquia. Quando usados sem propósito, só distraem e podem prejudicar acessibilidade.

📱
Mobile first Uma boa prática é começar pensando na experiência em telas menores e depois expandir o layout para tablets e desktops.

3.6 Sequência de refinamento que vale para quase todo projeto

  1. Versão 1: deixar o HTML legível, com espaçamento e cores básicas.
  2. Versão 2: criar componentes repetíveis, como botões, cards e blocos de destaque.
  3. Versão 3: ajustar layout com Flexbox ou Grid para diferentes larguras de tela.
  4. Versão 4: inserir fundos, imagens e microefeitos apenas onde agregam valor.
  5. Versão 5: revisar contraste, foco, peso visual e desempenho.
✨
Design forte nasce de repetição consciente Bons projetos de iniciante não dependem de dezenas de efeitos. Eles combinam poucas escolhas consistentes: paleta clara, tipografia estável, bom espaçamento e componentes reutilizados.

Exercícios do módulo

1. O que é o box model?

É o modelo de caixa do CSS formado por conteúdo, padding, border e margin.

2. Qual a diferença prática entre Flexbox e Grid?

Flexbox é ótimo para alinhamentos em uma dimensão (linha ou coluna). Grid é mais indicado para layouts em duas dimensões, com linhas e colunas.

3. Por que unidades relativas costumam ser preferidas em interfaces responsivas?

Porque se adaptam melhor a diferentes tamanhos de tela e configurações de acessibilidade, como aumento de fonte.
Módulo 4

JavaScript: lógica, DOM e eventos

JavaScript traz vida à interface. Com ele, a página deixa de ser apenas apresentação e passa a responder ao usuário.

🧠
Pensamento computacional Antes de programar, descreva o problema em etapas: entrada, processamento, decisão e saída. Essa lógica vale para qualquer linguagem.

4.1 Fundamentos que todo iniciante precisa dominar

Variáveis e tipos

let cria variáveis que podem mudar e const mantém a mesma referência. Entre os tipos mais comuns estão string, number, boolean, array e object.

Condições

if, else e operadores lógicos permitem decisões com base em critérios.

Funções

Blocos reutilizáveis que recebem dados, processam e retornam resultado.

Eventos

Respostas a ações como clique, digitação, envio de formulário ou rolagem.

4.2 DOM: a página vista como árvore

O DOM (Document Object Model) representa os elementos HTML como uma árvore de nós. JavaScript pode ler, alterar, criar ou remover esses elementos.

const botao = document.querySelector('#trocarTema');
const corpo = document.body;

botao.addEventListener('click', () => {
  corpo.classList.toggle('tema-escuro');
});

4.3 Exemplo real: botão que altera conteúdo

<button id="mostrarMensagem">Saudar turma</button>
<p id="saida"></p>

  const botao = document.querySelector('#mostrarMensagem');
  const saida = document.querySelector('#saida');

  botao.addEventListener('click', () => {
    saida.textContent = 'Bem-vindos ao laboratório de desenvolvimento web!';
  });
</script>

4.4 JavaScript moderno no front-end

Conceito Por que aparece no mercado Aplicação prática
async/await Facilita leitura de fluxos assíncronos e integração com APIs Buscar dados de eventos, produtos, usuários ou notícias
fetch() É a base de muitos projetos que consomem serviços web Carregar cards de uma API pública ou enviar dados de formulário
Tratamento de erro Interfaces profissionais precisam reagir quando algo falha Exibir mensagem amigável se a requisição não funcionar
Manipulação de estado Ajuda a manter a interface consistente após filtros, cliques e envios Atualizar listas, esconder painéis ou alternar tema 
async function carregarOficinas() {
  const lista = document.querySelector('#oficinas');

  try {
    const resposta = await fetch('https://jsonplaceholder.typicode.com/posts?_limit=3');
    const dados = await resposta.json();

    lista.innerHTML = dados.map((item) => `<li>${item.title}</li>`).join('');
  } catch (erro) {
    lista.innerHTML = '<li>Não foi possível carregar as oficinas agora.</li>';
  }
}

Leitura profissional: quem domina DOM, eventos, funções, tratamento de erros, controle básico de estado e integração com APIs já consegue desenvolver páginas dinâmicas muito próximas do que aparece em vagas de estágio e júnior.

Exercícios do módulo

1. O que significa dizer que o DOM é uma árvore?

Significa que os elementos da página são organizados em relação de pai, filho e irmãos, permitindo navegação e manipulação estruturada.

2. Para que serve addEventListener?

Serve para registrar uma função que será executada quando um evento acontecer, como clique, teclado ou envio de formulário.

3. Diferencie const e let.

const mantém a mesma referência; let permite reatribuição. Em objetos e arrays com const, o conteúdo interno ainda pode mudar.

Formulários, integração e experiência do usuário

Formulários conectam a interface aos dados. Eles aparecem em cadastros, pesquisas, login, compra, inscrição e suporte.

5.1 Estrutura de um formulário bem construído

<form id="contato" novalidate>
  <label for="nome">Nome</label>
  <input id="nome" name="nome" type="text" required>

  <label for="email">E-mail</label>
  <input id="email" name="email" type="email" required>

  <label for="mensagem">Mensagem</label>
  <textarea id="mensagem" name="mensagem" rows="5"></textarea>

  <button type="submit">Enviar</button>
</form>

5.2 Boas práticas

Prática Por que importa Exemplo
Rótulos claros Evita erro de preenchimento e melhora acessibilidade “Telefone com DDD” é melhor que apenas “Telefone”
Feedback imediato Ajuda o usuário a corrigir erros rapidamente Mensagem abaixo do campo inválido
Validação dupla Cliente ajuda UX, servidor garante segurança Checar e-mail no front-end e novamente no back-end
Botão com ação explícita Evita ambiguidade “Salvar inscrição” em vez de “OK”
🔐
Segurança importante Validar no navegador melhora a experiência, mas nunca substitui validação no servidor. Dados vindos do usuário podem ser inválidos, maliciosos ou incompletos.

5.3 Mini fluxo de validação com JavaScript

const form = document.querySelector('#contato');
const email = document.querySelector('#email');

form.addEventListener('submit', (evento) => {
  if (!email.value.includes('@')) {
    evento.preventDefault();
    alert('Digite um e-mail válido.');
  }
});

Observação: esse exemplo usa uma checagem simplificada apenas para fins didáticos. Em projetos reais, a validação de e-mail precisa ser mais criteriosa no cliente e também no servidor, porque a checagem feita no navegador pode ser contornada e não protege contra entradas maliciosas.

5.4 Tipos de campo, métodos de envio e iframes

Recurso Quando usar Exemplo prático
type="tel", date, number Quando o navegador pode ajudar com teclado e validação adequados Telefone, data de nascimento, quantidade
checkbox e radio Escolhas múltiplas ou exclusivas Interesses do aluno, turno preferido
<select>, <datalist>, <output> Listas guiadas, sugestões e resultados calculados Escolha de curso, autocomplete de cidade, total de inscrição
method="get" e method="post" GET para consultas; POST para envio de dados mais sensíveis ou extensos Busca interna do site e formulário de cadastro
<iframe> Conteúdo externo complementar Mapa da escola, vídeo tutorial, formulário publicado em outro serviço 
<form action="/busca" method="get">
  <label for="termo">Pesquisar</label>
  <input id="termo" name="q" type="search">
  <button type="submit">Buscar</button>
</form>

<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="Vídeo de apoio"
  loading="lazy"
  allowfullscreen>
</iframe>
🧾
Decisão rápida Se o formulário só consulta informações, GET costuma ser suficiente. Se ele grava dados, faz login ou envia informações pessoais, POST é a escolha mais adequada. Em iframe, revise origem, permissões e impacto no carregamento.

5.5 Integração com APIs, privacidade e confiança

Em projetos profissionais, formulários normalmente enviam dados para APIs, planilhas automatizadas, CRMs, bancos de dados ou plataformas de autenticação. Isso exige clareza sobre o que será coletado, por que será coletado e como o usuário receberá retorno após o envio.

Entrada

Campos claros, obrigatoriedade objetiva e orientação para evitar erros de preenchimento.

Processamento

Validação no cliente, validação no servidor e tratamento de erros sem expor detalhes técnicos ao usuário.

Retorno

Mensagem de sucesso, protocolo ou instrução de próximo passo para gerar confiança na experiência.

🛡️
Boa prática profissional Evite solicitar dados desnecessários. Em formulários escolares, peça apenas o que será realmente utilizado e explique como as informações serão tratadas.

6.3 Git, GitHub e publicação passo a passo

Uma rotina muito usada em cursos introdutórios é criar o projeto localmente, versionar cada etapa e depois publicar uma versão navegável. Esse fluxo ajuda a organizar o portfólio e a demonstrar evolução.

git branch -M main

  • Faça commits pequenos, por exemplo: estrutura HTML, estilos iniciais, validação do formulário.

6.5 Checklist de publicação que passa confiança

  • Revise ortografia, títulos, links quebrados e textos de botão antes de publicar.

  • Teste no celular, no desktop e com zoom aumentado para observar responsividade.
  • Verifique no DevTools se não há erros de console causados por scripts ou caminhos incorretos.
  • Publique a versão final apenas depois de um último commit com mensagem objetiva.

Ritual simples de revisão em dupla

  1. Um estudante navega e o outro observa erros de layout, ortografia e acessibilidade.
  2. Os dois registram três pontos fortes e três melhorias concretas.
  3. Depois da correção, publiquem de novo e comparem a nova versão com a anterior.

  1. Crie um repositório para um projeto simples da turma.
  2. Faça pelo menos três commits com mensagens descritivas.
  3. Publique a página em GitHub Pages, Netlify ou Vercel.
  4. Peça a um colega para testar o site e registrar um feedback curto.

Síntese final

Desenvolvimento web é uma área acessível para começar e profunda o bastante para construir carreira.

Ao dominar HTML, CSS e JavaScript, o estudante passa a compreender como páginas são estruturadas, estilizadas e programadas. Mais importante ainda: aprende a resolver problemas reais por meio de interfaces que comunicam informação com clareza.

Em um curso técnico, a meta não é apenas memorizar tags e comandos, mas construir autonomia. Isso envolve pesquisar documentação, testar hipóteses, corrigir erros, versionar o trabalho e apresentar soluções funcionais.

📌
Mensagem para a turma Não espere “saber tudo” para começar um projeto. A prática consistente transforma curiosidade em competência técnica.
