Módulo 1

O que é programação, algoritmo e Portugol

Antes de escrever código, é preciso entender como transformar um problema em uma sequência de passos.

🧠
Ideia central Programar não é decorar comandos. Programar é organizar raciocínio, testar hipóteses e construir soluções que possam ser executadas por uma máquina.

1.1 O que é um algoritmo

Um algoritmo é uma sequência lógica, finita e ordenada de instruções usada para resolver um problema. Ele precisa ter início, meio e fim. Também deve ser claro o suficiente para que outra pessoa — ou o computador — consiga executar os passos corretamente.

Exemplo cotidiano: fazer café. Você separa os materiais, aquece a água, coloca o pó no filtro, despeja a água e espera a bebida ficar pronta. Isso já é um algoritmo.

1.2 O que é programação

Programação é o processo de escrever instruções em uma linguagem que o computador consiga interpretar ou transformar em ações. Quando programamos, estamos descrevendo um algoritmo com precisão.

ConceitoDefiniçãoExemplo prático
ProblemaSituação que precisa de solução.Calcular a média de um aluno.
AlgoritmoPassos para resolver o problema.Ler notas, somar, dividir por 2 e mostrar resultado.
ProgramaAlgoritmo escrito em linguagem de programação.Código em Portugol que calcula a média.
TesteVerificação de que a solução funciona.Executar com notas diferentes.

1.3 Por que estudar Portugol

O Portugol é uma linguagem de ensino baseada em pseudocódigo estruturado. Sua principal vantagem é permitir que o estudante foque na lógica sem sofrer, logo no início, com detalhes complexos de linguagens profissionais.

Sintaxe em português

Comandos como leia, escreva e se tornam a lógica mais acessível.

Aprendizado gradual

Ajuda a dominar estruturas básicas antes de migrar para outras linguagens.

Foco em raciocínio

O aluno concentra energia em resolver problemas, não em decorar bibliotecas.

1.4 Como estudar programação de verdade

  • Ler o enunciado mais de uma vez e identificar entradas, processamento e saída.
  • Escrever a solução em passos curtos antes de codificar.
  • Executar com valores simples e depois com casos extremos.
  • Corrigir o raciocínio, não apenas o sintoma do erro.
  • Repetir exercícios parecidos até reconhecer padrões.

Exercícios de fixação

1. Explique com suas palavras a diferença entre problema, algoritmo e programa.

Problema é a situação a resolver; algoritmo é a sequência de passos para resolver; programa é o algoritmo escrito em uma linguagem que o computador executa.

2. Crie um algoritmo em linguagem natural para escovar os dentes.

Uma resposta possível: pegar a escova, colocar creme dental, escovar dentes superiores e inferiores, enxaguar a boca e guardar os materiais.

3. Cite duas vantagens de começar com Portugol antes de migrar para outra linguagem.

Facilita o entendimento da lógica em português e reduz a carga inicial de sintaxe complexa, permitindo foco em variáveis, condições e repetições.

Módulo 3

Estrutura de um programa em Portugol

Todo código precisa de organização. Entender a estrutura evita erros e melhora a leitura.

3.1 Anatomia básica

programa
{
    funcao inicio()
    {
        escreva("Meu primeiro programa")
    }
}
ParteFunção
programaMarca o início da definição do programa.
{ }Delimitam blocos de código.
funcao inicio()Ponto principal de execução.
escreva()Mostra informações para o usuário.

3.2 Sequência, clareza e indentação

O computador executa o programa na ordem em que os comandos aparecem, salvo quando existem estruturas de decisão ou repetição. Por isso, ordem importa. Também é importante manter o código bem identado, pois isso facilita leitura, manutenção e correção.

⚠️
Erro comum Iniciantes costumam abrir blocos e esquecer de fechar, ou colocar comandos fora da função principal. Sempre confira a correspondência entre chaves e a posição dos comandos.

3.3 Comentários e leitura de código

Comentários servem para explicar trechos importantes, registrar intenção e organizar o estudo. Mas comentário não substitui código claro. Primeiro escreva bem; depois, comente o que realmente ajuda.

programa
{
    funcao inicio()
    {
        // Exibe uma mensagem de boas-vindas
        escreva("Bem-vindo ao laboratório de algoritmos")
    }
}

Módulo 4

Variáveis, tipos de dados e operadores

Programas manipulam informações. Para isso, precisamos armazenar, atualizar e comparar valores.

4.1 O que é variável

Variável é um espaço nomeado de memória usado para guardar dados durante a execução do programa. Pense em uma variável como uma caixa com etiqueta: dentro dela você coloca um valor e pode trocar esse valor quando necessário.

programa
{
    funcao inicio()
    {
        inteiro idade = 16
        real altura = 1.72
        cadeia nome = "Marina"
        logico aprovado = verdadeiro
    }
}

4.2 Tipos de dados mais comuns

TipoUsado paraExemplo
inteiroNúmeros sem casas decimais.7, 15, -2
realNúmeros com parte decimal.8.5, 3.14
cadeiaTextos."Ana"

4.3 Operadores

Aritméticos

Usados para cálculos numéricos.

Relacionais

==, !=, >, <, >=, <=

Usados para comparar valores.

e, ou, nao

Usados para combinar condições.

Atribuição

=

Usado para guardar valor em variável.

4.4 Expressões e precedência

Quando uma expressão possui vários operadores, o programa segue uma ordem de avaliação. Parênteses ajudam a deixar o cálculo mais claro e menos sujeito a erro.

inteiro resultado = (8 + 2) * 3
// resultado será 30
📌
Nomeie bem suas variáveis Prefira mediaFinal a x. Um nome claro reduz erros e melhora a manutenção do código.

Módulo 5

Entrada, saída e construção de interação

Um programa útil normalmente recebe dados do usuário, processa e devolve uma resposta.

5.1 Comandos de saída

Usamos escreva para mostrar mensagens, números e resultados ao usuário. Uma boa saída é clara, objetiva e ajuda quem está usando o programa a entender o que aconteceu.

5.2 Comandos de entrada

Usamos leia para capturar informações digitadas pelo usuário. Sempre explique o que deve ser digitado antes da leitura.

programa
{
    funcao inicio()
    {
        cadeia nome
        inteiro idade

        escreva("Digite seu nome: ")
        leia(nome)
        escreva("Digite sua idade: ")
        leia(idade)

        escreva("Olá, ", nome, "! Você tem ", idade, " anos.")
    }
}

5.3 Modelo E-P-S

Uma forma profissional de pensar programas é separar o problema em três partes:

  • Entrada: o que o programa precisa receber.
  • Processamento: o que o programa faz com os dados.
  • Saída: o que o programa devolve.
EntradaNotas do aluno
ProcessamentoSoma e divisão
SaídaMédia final

5.4 Exemplo completo: média simples

programa
{
    funcao inicio()
    {
        real nota1, nota2, media

        escreva("Digite a primeira nota: ")
        leia(nota1)
        escreva("Digite a segunda nota: ")
        leia(nota2)

        media = (nota1 + nota2) / 2

        escreva("Média final = ", media)
    }
}
🎯
Meta de qualidade Um bom programa inicial orienta o usuário, evita mensagens confusas e mostra resultados com contexto. Em vez de mostrar apenas 7.5, mostre Média final = 7.5.

Exercícios de interação

1. Faça um programa que leia o nome e a cidade de um aluno e mostre uma frase completa.

escreva("Nome: ")
leia(nome)
escreva("Cidade: ")
leia(cidade)
escreva(nome, " mora em ", cidade)

2. Crie um programa que leia largura e comprimento de um terreno e mostre a área.

A lógica é: ler largura, ler comprimento, calcular área multiplicando os dois valores e mostrar a área.

3. Modifique o programa da média para ler três notas em vez de duas.

Declare mais uma variável, some as três notas e divida por 3.
Módulo 6

Estruturas condicionais

Nem todo programa segue a mesma rota. Muitas vezes ele precisa decidir.

6.1 Decisão simples

A estrutura se permite executar um bloco somente quando uma condição for verdadeira.

se (media >= 6)
{
    escreva("Aprovado")
}

6.2 Decisão composta

Quando há dois caminhos possíveis, usamos se e senao.

se (media >= 6)
{
    escreva("Aprovado")
}
senao
{
    escreva("Reprovado")
}

6.3 Decisão encadeada

Quando existem várias faixas ou categorias, podemos encadear condições.

se (nota >= 9)
{
    escreva("Excelente")
}
senao se (nota >= 7)
{
    escreva("Bom")
}
senao se (nota >= 6)
{
    escreva("Regular")
}
senao
{
    escreva("Precisa melhorar")
}

6.4 Operadores lógicos em condições

Podemos combinar critérios. Exemplo: para aprovação, talvez o aluno precise de média mínima e frequência suficiente.

se (media >= 6 e frequencia >= 75)
{
    escreva("Aprovado")
}
senao
{
    escreva("Situação irregular")
}
🧩
Pense antes de codificar Em condicionais, desenhe os casos possíveis em papel. Isso evita lacunas como esquecer um intervalo de notas ou criar regras contraditórias.

Exercícios com condições

1. Faça um programa que leia a idade de uma pessoa e informe se ela é maior de idade.

inteiro idade
escreva("Idade: ")
leia(idade)
se (idade >= 18)
{
    escreva("Maior de idade")
}
senao
{
    escreva("Menor de idade")
}

2. Leia um número e informe se ele é positivo, negativo ou zero.

Use três casos: maior que zero, menor que zero e igual a zero.

3. Crie um programa que leia duas notas e a frequência. Informe “aprovado”, “recuperação” ou “reprovado”.

Uma estratégia possível: média maior ou igual a 6 e frequência suficiente = aprovado; média entre 4 e 5,9 = recuperação; abaixo disso = reprovado.
Módulo 7

Estruturas de repetição

Quando precisamos executar instruções várias vezes, usamos laços de repetição.

7.1 Por que repetir

Imagine somar a nota de 30 alunos digitando o mesmo bloco 30 vezes. Isso seria improdutivo e sujeito a erro. Laços tornam o código mais enxuto, escalável e profissional.

7.2 Laço para

É ideal quando sabemos quantas vezes queremos repetir.

inteiro i
para (i = 1; i <= 5; i++)
{
    escreva(i, "\n")
}

7.3 Laço enquanto

É útil quando a repetição depende de uma condição.

inteiro senha = 0
enquanto (senha != 1234)
{
    escreva("Digite a senha: ")
    leia(senha)
}

7.4 Cuidados com repetição infinita

Se a condição nunca deixar de ser verdadeira, o programa ficará preso em loop. Por isso, em cada repetição, é preciso atualizar as variáveis de controle corretamente.

EstruturaMelhor usoExemplo
paraQuantidade conhecida de repetições.Mostrar números de 1 a 10.
enquantoRepetir até uma condição mudar.Pedir senha até acertar.
faca...enquantoGarantir ao menos uma execução.Menu que precisa aparecer pelo menos uma vez.

Laboratório guiado

  1. Mostre os números de 1 a 20.
  2. Mostre apenas os pares entre 2 e 20.
  3. Some os números de 1 a 100.
  4. Leia 5 notas e calcule a média da turma.

Exercícios com laços

1. Escreva um programa que mostre a tabuada do 7.

Use um laço para de 1 até 10 e mostre 7 * i em cada repetição.

2. Faça um programa que leia 10 números e informe quantos são positivos.

Use um contador iniciado em zero. A cada número positivo lido, some 1 ao contador.

3. Crie um programa que peça uma senha até que o valor correto seja digitado.

inteiro senha = 0
enquanto (senha != 1234)
{
    escreva("Digite a senha: ")
    leia(senha)
}
escreva("Acesso liberado")
Módulo 8

Vetores, matrizes e organização de dados

Quando lidamos com vários valores relacionados, estruturas indexadas tornam o algoritmo mais eficiente.

8.1 Vetores

Vetores armazenam vários valores do mesmo tipo sob um único nome, acessados por posição. São úteis para notas, preços, nomes, medições e contagens.

inteiro numeros[5]

numeros[0] = 10
numeros[1] = 20
numeros[2] = 30
numeros[3] = 40
numeros[4] = 50

8.2 Percorrendo vetores

inteiro i, soma = 0
inteiro notas[5]

para (i = 0; i < 5; i++)
{
    escreva("Digite a nota ", i + 1, ": ")
    leia(notas[i])
    soma = soma + notas[i]
}

8.3 Matrizes

Matrizes são tabelas com linhas e colunas. Elas são úteis para horários, mapas, planilhas simples, jogos, assentos ou dados organizados em grade.

inteiro tabuleiro[3][3]

8.4 Situações reais de uso

Vetores

Notas de uma turma, temperaturas do dia, estoque de peças.

Matrizes

Mapa de sala, horário semanal, grade de sensores.

Índices

Permitem acessar um item específico com rapidez.

🔢
Atenção aos índices Muitos erros em vetores e matrizes acontecem porque o aluno tenta acessar uma posição que não existe. Revise sempre o limite inicial e final do laço.

Exercícios de estruturas de dados

1. Leia 5 números para um vetor e mostre o maior valor digitado.

Guarde o primeiro valor como maior inicial e compare com os demais durante o laço.

2. Preencha uma matriz 2x2 com números e mostre todos os elementos.

Use dois laços aninhados: um para linhas e outro para colunas.

3. Crie um programa que leia 6 notas em vetor e conte quantas são maiores ou iguais a 6.

Use um contador de aprovados; para cada posição do vetor, teste se a nota é maior ou igual a 6.
Módulo 9

Funções, procedimentos e modularização

Programas profissionais são divididos em partes menores, reutilizáveis e fáceis de testar.

9.1 Por que modularizar

Quando o programa cresce, colocar tudo em um único bloco torna a manutenção difícil. Dividir responsabilidades em funções e procedimentos melhora legibilidade, reaproveitamento e teste.

9.2 Função que retorna valor

funcao real calcularMedia(real n1, real n2)
{
    retorne (n1 + n2) / 2
}

9.3 Procedimento

Um procedimento executa uma tarefa, mas não precisa devolver resultado diretamente.

funcao mensagemBoasVindas()
{
    escreva("Bem-vindo ao sistema!\n")
}

9.4 Exemplo integrado

programa
{
    funcao real calcularMedia(real n1, real n2)
    {
        retorne (n1 + n2) / 2
    }

    funcao inicio()
    {
        real a, b, media
        escreva("Nota 1: ")
        leia(a)
        escreva("Nota 2: ")
        leia(b)

        media = calcularMedia(a, b)
        escreva("Média = ", media)
    }
}
🧱
Pensamento modular Sempre que um trecho fizer uma tarefa específica e puder ser reutilizado, pense em transformá-lo em função. Isso é um passo importante rumo ao desenvolvimento profissional.

Exercícios de modularização

1. Crie uma função que receba um número e retorne seu dobro.

funcao inteiro dobro(inteiro n)
{
    retorne n * 2
}

2. Crie uma função que receba base e altura e retorne a área do retângulo.

A função deve receber dois valores reais e retornar o produto deles.

3. Transforme um programa que calcula média em duas partes: leitura no início e cálculo em função separada.

Esse exercício treina separação de responsabilidades: a função calcula, a função principal interage com o usuário.
Módulo 10

Estratégia algorítmica, teste e depuração

Escrever código é só uma parte do trabalho. Pensar, testar e corrigir faz parte do processo profissional.

10.1 Método recomendado para resolver exercícios

  1. Leia o problema até entender o objetivo sem pressa.
  2. Identifique entradas, processamento e saída.
  3. Escreva um plano em português.
  4. Codifique a solução básica.
  5. Teste com casos simples, extremos e inesperados.
  6. Refatore nomes e organização.

10.2 Tipos de erro

Tipo de erroDescriçãoExemplo
SintaxeComando escrito de forma incorreta.Esquecer parêntese ou chave.
ExecuçãoO programa roda, mas falha em determinada situação.Divisão por zero.
LógicaO programa executa, porém entrega resultado errado.Somar quando deveria multiplicar.

10.3 Teste de mesa

O teste de mesa consiste em simular o programa manualmente, anotando o valor das variáveis a cada etapa. É uma técnica poderosa para compreender laços, decisões e funções.

Exemplo de teste de mesa

Considere o algoritmo abaixo:

inteiro x = 2
inteiro y = 3
inteiro z = x + y * 2

Valores: x = 2, y = 3, z = 8.

10.4 Checklist de revisão antes de entregar

  • O programa pede todas as entradas necessárias?
  • Os nomes das variáveis fazem sentido?
  • O cálculo principal está correto?
  • As condições cobrem todos os casos?
  • Os laços terminam corretamente?
  • A saída está clara para o usuário?

Exercícios de análise

1. Um programa deveria calcular média, mas somou as notas e esqueceu de dividir. Que tipo de erro é esse?

Erro de lógica, porque o programa executa, mas o resultado está errado.

2. Faça um teste de mesa para um laço que mostra os números de 1 a 3.

Monte uma tabela com a variável de controle em cada passo: 1, 2, 3, e observe quando a condição deixa de ser verdadeira.

3. Analise um programa seu e escreva um pequeno relatório: qual era o problema, qual erro encontrou e como corrigiu.

O foco é desenvolver pensamento reflexivo e documentação técnica do processo de correção.
Módulo 11

Projeto integrador e trilha de evolução

Chegou a hora de unir os conceitos em um projeto completo e construir um plano de continuidade.

11.1 Projeto sugerido: sistema de boletim escolar

Desenvolva um programa que:

  • Leia o nome do aluno.
  • Leia 4 notas.
  • Calcule a média.
  • Informe situação: aprovado, recuperação ou reprovado.
  • Permita repetir o cadastro de vários alunos.
  • Ao final, mostre quantos alunos foram aprovados.

11.2 Competências mobilizadas

Entrada e saída

Coleta e apresentação organizada de informações.

Cálculo

Uso de variáveis e operadores.

Condições

Classificação por faixas de resultado.

Repetição

Cadastro de vários alunos.

Vetores

Opcional para armazenar turmas maiores.

Funções

Separação do cálculo e da exibição.

Desafios finais

1. Implemente o sistema de boletim escolar completo.

Comece pela versão para um aluno. Depois acrescente laço de repetição para vários alunos. Por fim, conte os aprovados.

2. Faça um sistema de caixa simples que some preços até o usuário decidir encerrar.

Entrada, repetição, acumulador, decisão e saída formatada.

3. Projete um menu com três opções: calcular média, calcular área e sair.

A.1 Glossário rápido

Algoritmo

Sequência lógica de passos para resolver um problema.

Variável

Espaço usado para armazenar um valor durante a execução.

Condição

Teste lógico que decide entre caminhos possíveis.

Laço

Estrutura que repete comandos.

Vetor

Conjunto indexado de valores do mesmo tipo.

Função

Bloco reutilizável que pode retornar resultado.
